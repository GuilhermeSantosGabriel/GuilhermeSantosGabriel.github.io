**This series of posts serve to show my experiences following the "1st Phase: Linux kernel cycle" of the Free Software Development Course made by FLOSS at USP.**

Today i start this journey by setting up the test environment using QEMU and libvirt! I am using a fedora 41 + kde, just as a heads up.

Today's tutorial: https://flusp.ime.usp.br/kernel/qemu-libvirt-setup/

### What i did

The tutorial basically let me set up a QEMU libvirt environment. We made a directory for the vm files and configured access for QEMU. In my case i had to actually create the QEMU user and group because for some reason my fedora installation didn't do it by default. We inspected a base image (which i had to download a different one than the provided in the tutorial because the link was down), expanded the image and got the kernel/initrd (the part that is provided to a boot loader so it loads our kernel into the ram) in a different place.

### Problems

The only problem i had was the stuff related to the QEMU user and group. I had help from David de Barros (from FLOSS at USP, thank you so much!) and then everything was good to go.
